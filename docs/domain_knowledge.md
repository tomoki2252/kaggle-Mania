# ドメイン知識 - NCAA大学バスケットボールとMarch Madness

本ドキュメントでは、EDA・モデリングを行う上で必要なバスケットボールとNCAAトーナメントの背景知識をまとめる。

---

## 1. バスケットボールの基本ルール

### 試合の仕組み

- **5人 vs 5人**で行うチームスポーツ
- 試合時間は**前半20分 + 後半20分 = 40分**（大学バスケの場合。NBAは48分）
- 時間内により多くの得点を取ったチームが勝利
- 同点の場合は**延長戦（Overtime, OT）**: 5分間の延長を決着がつくまで繰り返す
  - データ上は `NumOT` で延長回数が記録される（0=延長なし、1=1回延長...最大6回まで実績あり）
  - **引き分けは存在しない**。必ずどちらかが勝つ

### 得点方法

| シュートの種類 | 得点 | 説明 |
|--------------|------|------|
| フィールドゴール（FG） | 2点 | 3ポイントライン内からのシュート |
| 3ポイントシュート（FG3） | 3点 | 3ポイントラインの外側からのシュート |
| フリースロー（FT） | 1点 | ファウルを受けた際に与えられる。ラインから無防備で打てる |

- データの `WScore` / `LScore` は各チームの合計得点
- 男子レギュラーシーズンの平均スコア: 勝者 約77点、敗者 約65点

### 主要なボックススコア統計（DetailedResultsの理解に必要）

| 統計 | データ上の略称 | 説明 |
|------|--------------|------|
| フィールドゴール成功数/試投数 | FGM / FGA | 2点+3点シュートの成功数と打った回数 |
| 3ポイント成功数/試投数 | FGM3 / FGA3 | 3ポイントシュートの成功数と打った回数 |
| フリースロー成功数/試投数 | FTM / FTA | フリースローの成功数と打った回数 |
| オフェンスリバウンド | OR | 自チームのシュートが外れた後に自チームが確保した回数 |
| ディフェンスリバウンド | DR | 相手のシュートが外れた後に自チームが確保した回数 |
| アシスト | Ast | 得点につながるパスの回数 |
| ターンオーバー | TO | ミスで相手にボールを渡してしまった回数 |
| スティール | Stl | 相手からボールを奪った回数 |
| ブロック | Blk | 相手のシュートをはたき落とした回数 |
| パーソナルファウル | PF | 身体接触による反則の回数 |

### バスケットボール分析の重要指標（Four Factors）

バスケットボール統計学者 Dean Oliver が提唱した、試合の勝敗を決定する4大要因。DetailedResultsから算出可能。

| 指標 | 計算式 | 意味 |
|------|--------|------|
| **eFG%**（実効FG%） | (FGM + 0.5 × FGM3) / FGA | 3ポイントの追加価値を考慮したシュート効率 |
| **TO%**（ターンオーバー率） | TO / (FGA + 0.475 × FTA + TO) | ポゼッション（攻撃機会）あたりのミスの頻度 |
| **OR%**（オフェンスリバウンド率） | OR / (OR + 相手DR) | シュートを外した後の二次攻撃機会の獲得率 |
| **FT Rate**（フリースロー率） | FTM / FGA | フリースローによる得点の効率 |

> **EDAでの活用**: 各チームのシーズン平均Four Factorsを算出し、トーナメント勝敗との相関を確認することで、どの指標が予測に有効かを判断できる。

## 2. NCAA Division I バスケットボール

### NCAAとは

- **NCAA（National Collegiate Athletic Association）**: 全米大学体育協会。米国の大学スポーツを統括する組織
- Division I（D1）は最も競技レベルが高い区分で、男子約360チーム、女子約370チームが所属
- 各チームは「**カンファレンス**」と呼ばれるリーグに属する

### カンファレンスの仕組み

カンファレンスは地理的・歴史的な理由でグループ化された大学のリーグ。

**強豪カンファレンス（Power Conference）** — トーナメント出場チームを多く輩出:
- **SEC**（Southeastern Conference）
- **Big Ten**
- **ACC**（Atlantic Coast Conference）
- **Big 12**
- **Big East**

**中小カンファレンス（Mid-Major）**:
- 上記以外の約30のカンファレンス
- チーム数は少なく、トーナメント出場枠は通常1（カンファレンストーナメント優勝チームのみ）

> **EDAでの活用**: Power Conferenceのチームは対戦相手のレベルも高いため勝率が低く見えることがある。単純な勝率だけでなく、対戦相手の強さ（Strength of Schedule）を考慮する必要がある。

### シーズンの流れ

```
11月        12月        1月         2月         3月            4月
|-----------|-----------|-----------|-----------|--------------|
  非カンファ戦     カンファレンス戦中心        カンファ   NCAA
  (プレシーズン)   (リーグ戦)               トーナメント トーナメント
                                              ↑          ↑
                                          Selection   Final Four
                                           Sunday     → 優勝決定
```

データ上のタイムライン（DayNum基準）:
| DayNum | 期間 | 内容 |
|--------|------|------|
| 0-132 | 11月〜3月上旬 | レギュラーシーズン（約30試合/チーム） |
| 133頃 | 3月中旬 | Selection Sunday（トーナメント出場チーム発表） |
| 134-135 | 3月中旬 | First Four（プレイインゲーム） |
| 136-137 | 3月下旬 | 1回戦（Round of 64） |
| 138-139 | 3月下旬 | 2回戦（Round of 32） |
| 143-144 | 3月末 | Sweet 16（地区準決勝） |
| 145-146 | 3月末 | Elite 8（地区決勝） |
| 152 | 4月上旬 | Final Four（準決勝） |
| 154 | 4月上旬 | Championship（決勝） |

## 3. NCAAトーナメント（March Madness）

「March Madness」とも呼ばれ、米国で最も人気のあるスポーツイベントの一つ。

### 出場チーム数

| 期間 | 男子 | 女子 |
|------|------|------|
| 〜2010 | 64〜65チーム | 64チーム |
| 2011〜2021 | 68チーム | 64チーム |
| 2022〜 | 68チーム | 68チーム |

### 出場チームの選出方法

68チームの内訳:
1. **自動出場枠（Automatic Bid）**: 約32枠。各カンファレンストーナメントの優勝チームに与えられる
2. **一般選出枠（At-Large Bid）**: 約36枠。NCAA選考委員会がシーズン成績を総合評価して選出

> **EDAでの活用**: 自動出場枠で入るチーム（特に中小カンファレンスの優勝チーム）は、一般選出枠のチームと比べて実力差が大きい場合がある。

### シード制度

選出された68チームは4つの**地域（Region）** に分けられ、各地域内で**1〜16のシード番号**が付与される。

- データ上の表現: `W01`, `X16`, `Y08` など（地域文字 + シード番号）
- **W/X/Y/Z** はデータ上の地域識別子で、実際の地域名（East, West, Midwest, South等）は `Seasons.csv` の `RegionW`〜`RegionZ` で確認できる
- シード1が最強、シード16が最弱

**シード番号の意味**:
| シード | 一般的な特徴 |
|--------|-------------|
| 1-4 | 強豪。Power Conference上位チームや中小カンファレンスの圧倒的優勝チーム |
| 5-8 | 中堅上位。勝ち残る力はあるが安定感にやや欠ける |
| 9-12 | 中堅。初戦の勝敗が拮抗する（特に5 vs 12、8 vs 9は接戦が多い） |
| 13-16 | 格下。中小カンファレンスの自動出場枠が多い |

> **EDAでの重要ポイント**: シード差と勝率の関係は非線形。1 vs 16 の勝率は約99%だが、8 vs 9 は約50%。この「S字カーブ」をうまく捉えるモデルが高精度になる。

### プレイインゲーム（First Four）

68チームを64チームに絞るための予備戦。4試合が行われる。

- データ上はシード末尾に `a` / `b` が付く（例: `W16a`, `W16b`）
- `a` と `b` のチームが対戦し、勝者が本戦の `W16` として進出
- 通常、16シード同士×2試合、11シード同士×2試合が行われる

### トーナメントブラケット（対戦表）の構造

```
地域W                                    地域X
W01 ─┐                                  X01 ─┐
W16 ─┘─┐                                X16 ─┘─┐
W08 ─┐  │                               X08 ─┐  │
W09 ─┘─┘─┐                              X09 ─┘─┘─┐
W05 ─┐    │                              X05 ─┐    │
W12 ─┘─┐  │                              X12 ─┘─┐  │
W04 ─┐  │ │                              X04 ─┐  │ │
W13 ─┘─┘─┘─── 地域W勝者 ─┐              X13 ─┘─┘─┘─── 地域X勝者 ─┐
                          ├── 決勝 ──                              │
地域Y勝者 ─┘              地域Z勝者 ─┘
   (同様の構造)               (同様の構造)
```

- **Round of 64**（1回戦）: 各地域で1 vs 16、2 vs 15、...、8 vs 9の8試合（計32試合）
- **Round of 32**（2回戦）: 1回戦の勝者同士が対戦（計16試合）
- **Sweet 16**（地区準決勝）: 各地域4チームずつ（計8試合）
- **Elite 8**（地区決勝）: 各地域2チームずつ（計4試合）
- **Final Four**（準決勝）: 地域勝者4チームのうち W vs X、Y vs Z（計2試合）
- **Championship**（決勝）: 1試合

合計: 4 (First Four) + 32 + 16 + 8 + 4 + 2 + 1 = **67試合**（68チームの場合）

> `MNCAATourneySlots.csv` の `Slot` 列はこの構造をコード化している:
> - `R1W1` = Round 1, 地域W, 第1対戦（W01 vs W16）
> - `R2W1` = Round 2, 地域W, 第1対戦（R1W1勝者 vs R1W8勝者）
> - `R5WX` = Round 5, W地域勝者 vs X地域勝者（準決勝）
> - `R6CH` = Round 6, Championship（決勝）

### 男女の違い（データに影響する重要な差異）

| 項目 | 男子 | 女子 |
|------|------|------|
| トーナメント会場 | **全試合中立地** (WLoc=N) | **上位シードがホーム開催**の場合あり (H/A/N) |
| 出場チーム数 | 68（2011年〜） | 64→68（2022年から68に拡大） |
| MasseyOrdinals | あり（196種のランキング） | なし |
| コーチデータ | あり | なし |
| データ開始年 | 1985年〜 | 1998年〜 |

> **EDAでの重要ポイント**: 女子トーナメントではホーム開催が存在するため、`WLoc` を特徴量に含めることが男子以上に重要。女子トーナメントの全1,717試合中、ホーム勝利(H)が676回、アウェイ勝利(A)が155回と大きな偏りがある（ホームチーム勝率 約81%）。

## 4. 予測モデリングに直結する統計的な知識

### シード差と勝率の関係（最重要）

NCAAトーナメントの勝率はシード番号の差に強く依存する。これが最も基本的な予測の土台になる。

| 対戦カード | シード差 | 歴史的な上位シード勝率（目安） |
|-----------|---------|------------------------------|
| 1 vs 16 | 15 | 約99% |
| 2 vs 15 | 13 | 約94% |
| 3 vs 14 | 11 | 約85% |
| 4 vs 13 | 9 | 約79% |
| 5 vs 12 | 7 | 約65% |
| 6 vs 11 | 5 | 約63% |
| 7 vs 10 | 3 | 約61% |
| 8 vs 9 | 1 | 約51% |

> **モデリングのヒント**: シード差→勝率の変換はロジスティック関数でよく近似される。シード差だけのモデルでもBrier score 0.15台を達成できることが多い。

### アップセット（番狂わせ）

低シードが高シードに勝つ「アップセット」はMarch Madnessの醍醐味。

- **5 vs 12** は最も有名なアップセットゾーン。12シードは強豪カンファレンスで好成績を残したが一般選出枠ギリギリのチーム vs 中小カンファレンスの優勝チームという構図が多く、拮抗しやすい
- **予測精度の鍵**: アップセットの確率を適切に見積もること。全部0.5に近づけすぎるとBrier scoreが悪化し、逆に確信度を上げすぎても外した場合のペナルティが大きい

### ホームコートアドバンテージ

- レギュラーシーズンでは各チームが自校の体育館（ホーム）で試合を行う
- データの `WLoc` 列: **H**（勝者がホーム）/ **A**（勝者がアウェイ）/ **N**（中立地）
- ホームチームは一般的に3-4点のアドバンテージがあるとされる
- **男子トーナメントは全試合中立地**なので、トーナメント予測ではこの要素の影響は小さい
- **女子トーナメントはホーム開催あり**なので、会場の影響を特徴量に含めるべき

### Strength of Schedule（SOS、対戦相手の強さ）

弱いチームとばかり対戦して勝率が高いチームと、強豪相手に勝ち抜いてきたチームでは、同じ勝率でも実力が異なる。

- 各チームの対戦相手の平均勝率を計算することでSOSを近似できる
- Power Conferenceのチームは対戦相手が強いためSOS値が高くなりやすい
- **RPI（Rating Percentage Index）** = 勝率25% + 対戦相手の勝率50% + 対戦相手の対戦相手の勝率25% として、公式にも使われてきた指標

### ポゼッション（攻撃機会）の概念

バスケットボールの分析では、得失点を「ポゼッション（1回の攻撃機会）あたり」で正規化するのが重要。

```
ポゼッション ≈ FGA - OR + TO + 0.475 × FTA
```

- **テンポ（試合のペース）**: 1試合あたりのポゼッション数。チームによって大きく異なる（60〜80ポゼッション程度）
- テンポが速いチーム同士の試合は高得点になり、遅いチーム同士は低得点になる
- **Offensive Efficiency** = 得点 / ポゼッション × 100（100ポゼッションあたりの得点）
- **Defensive Efficiency** = 失点 / ポゼッション × 100

> **EDAでの活用**: 単純な平均得点よりも、ポゼッションあたりの効率のほうがチームの真の強さを反映する。テンポの異なるチーム同士を公平に比較するために必須の概念。

## 5. このコンペ特有の注意点

### 全チームペアの予測

通常のトーナメント予測では68チーム間の対戦のみを予測するが、本コンペでは **D1全チーム間のあらゆる組み合わせ** を予測する必要がある。

- 男子: 約380チーム → 約72,000ペア
- 女子: 約380チーム → 約72,000ペア
- トーナメント不出場チーム同士のペアにも予測を出す必要がある（ただしスコアリングには使われない）

> **実務上の影響**: シードがないチーム（トーナメント不出場）の勝率予測には、レギュラーシーズンの統計やカンファレンス情報に頼る必要がある。

### Brier Score（評価指標）

```
Brier Score = (1/N) × Σ(予測確率 - 実際の結果)²
```

- 実際の結果: 下位TeamIDが勝った場合 = 1、負けた場合 = 0
- **小さいほど良い**（完全予測 = 0.0、ランダム = 0.25）
- 確率を正確に推定することが重要。「勝つか負けるか」のバイナリ予測ではなく、「どのくらいの確率で勝つか」の精度が問われる

> **モデリングへの影響**: Brier scoreはMSEと等価なので、モデルの出力がキャリブレーション（確率の正確さ）されていることが重要。例えば「70%勝つ」と予測した試合群の実際の勝率が本当に70%前後になっているか。

### 2026年の特殊性

- 2026年はまだレギュラーシーズン中（現在2026-03-16）
- トーナメントのシードはSelection Sunday（3月中旬）に発表される
- レギュラーシーズンデータは存在するが、トーナメント結果はまだない
- 提出は何度でも可能だが、最終的に1つの提出を自分で選ぶ必要がある
