# 評価指標と選定根拠

## 1. Brier Score (Primary)

### 定義

$$\text{Brier Score} = \frac{1}{N} \sum_{i=1}^{N} (p_i - o_i)^2$$

- $p_i$: 予測確率 (LowerTeamIDが勝つ確率)
- $o_i$: 実際の結果 (1 = LowerTeamIDが勝利, 0 = 敗北)
- $N$: 試合数

値域は [0, 1]。0が完璧な予測、1が最悪の予測。全試合0.5と予測した場合は0.25。

### 選定根拠

**コンペティションの公式評価指標**。提出された予測確率と実際の試合結果のMSE (Mean Squared Error) として定義されており、Brier Scoreと等価である。

Brier Scoreの特性:
- **確率の正確さ (calibration)** と **識別力 (resolution)** の両方を評価する
- 70%と予測して実際に勝った場合のペナルティ (0.09) は、99%と予測して負けた場合のペナルティ (0.98) より遥かに小さい → **過信に対して二次的に罰する**
- 確信度が高いほどリターンも大きいがリスクも大きい → 適切な不確実性の表現が求められる

### ベースライン比較

| 戦略 | Brier Score |
|------|-------------|
| 全試合0.5と予測 | 0.2500 |
| 勝率のみで予測 | ~0.2100 |
| Eloのみ (K=32) | ~0.1920 |
| **本モデル (v3)** | **0.1705** |
| 理論的下限 (完璧な予測) | 0.0000 |

> 全試合0.5予測から本モデルへの改善率: (0.25 - 0.1705) / 0.25 = **31.8%**

## 2. Log Loss (Secondary)

### 定義

$$\text{Log Loss} = -\frac{1}{N} \sum_{i=1}^{N} \left[ o_i \log(p_i) + (1 - o_i) \log(1 - p_i) \right]$$

### 選定根拠

LightGBMの学習目的関数 (`objective: binary`) がLog Loss (binary cross-entropy) を最小化するため、**モデルの最適化対象と一致する指標**として監視している。

Brier ScoreとLog Lossの違い:
- Log Lossは**極端な誤りにより強く罰する** (0.99と予測して負けた場合、Brier=0.98だがLog Loss=4.6)
- Brier Scoreは二次ペナルティ、Log Lossは対数ペナルティ
- 実務上、両者の最適化は高い相関を持つが、Log Lossの方がクリッピングの影響を受けやすい

### バリデーション結果

| 指標 | 値 |
|------|-----|
| Log Loss (single) | 0.506 |
| Log Loss (ensemble) | 0.507 |

> クリッピング [0.02, 0.98] を適用しているため、Log Lossの値はクリッピング無しの場合より高くなる。これはBrier Score最適化のためのトレードオフ。

## 3. Accuracy (Secondary)

### 定義

$$\text{Accuracy} = \frac{1}{N} \sum_{i=1}^{N} \mathbb{1}[\hat{y}_i = o_i]$$

- $\hat{y}_i = 1$ if $p_i > 0.5$, else $0$

### 選定根拠

直感的な理解のしやすさから補助指標として使用。「全試合の何%を正しく勝敗予測できたか」。

ただし、Accuracyには以下の限界がある:
- **確率の質を評価しない**: 0.51と予測しても0.99と予測しても、勝てばどちらも正解
- **接戦の重要性を無視**: 8 vs 9シードのような五分五分の試合を間違えても、1 vs 16のような確実な試合を間違えても、同じ1ミス
- コンペティションの評価には使えないが、モデルの「方向性」を確認するのに有用

### バリデーション結果

| 指標 | 値 |
|------|-----|
| Accuracy (single) | 74.4% |
| Accuracy (ensemble) | 74.9% |

参考: NCAAトーナメント予測の一般的なAccuracyは 70-75% とされる。

## 4. クリッピングの設計根拠

予測値を [0.02, 0.98] にクリッピングしている。

### なぜクリッピングするのか

- **Brier Scoreは二次ペナルティ**: 0.99と予測して負けた場合の損失 $(0.99-0)^2 = 0.98$ は、0.80と予測して勝った場合の損失 $(0.80-1)^2 = 0.04$ の約25倍
- 極端な確信が外れた場合のコストが非常に大きいため、確信度の上限を制限する
- NCAA トーナメントでは歴史的にアップセットが発生する (1 vs 16でも98.8%であり100%ではない)

### クリッピング範囲の選択

| 範囲 | 特性 |
|------|------|
| [0.01, 0.99] | 広い: 強い確信を許容、アップセット時のペナルティ大 |
| **[0.02, 0.98]** | **採用**: バランス型 |
| [0.05, 0.95] | 狭い: 安全だが、確実な試合での改善幅が小さい |

0.02-0.98 は「1 vs 16 のアップセット確率 (~1.2%)」と概ね一致しており、データに基づく妥当な範囲。

## 5. バリデーション設計の根拠

### 時系列分割 (2021-2025をテスト)

ランダム分割ではなく**時系列分割**を採用する理由:
- 実際の予測タスクは「過去のデータで未来のトーナメントを予測する」
- ランダム分割では未来の情報がリーク (data leakage) する可能性がある
- チームのEloや統計量は時系列で変化するため、時間方向の外挿性能を測る必要がある

### なぜ2021-2025の5年間か

- 2020年はCOVIDでトーナメント中止 → 自動的に除外
- 5年分 (665試合) はバリデーションに十分なサンプルサイズ
- 直近のデータほど2026年の予測に近い条件

### シーズン別スコアの安定性

| Season | Brier | 備考 |
|--------|-------|------|
| 2021 | 0.1890 | COVID明け、変則的なシーズン |
| 2022 | 0.1852 | |
| 2023 | 0.1897 | |
| 2024 | 0.1600 | アップセット少なめ |
| 2025 | 0.1302 | 予測しやすい年 |

年度間のBrierの幅 (0.13-0.19) は、トーナメントの固有の不確実性（アップセットの発生頻度）に起因する。モデルの問題ではなくタスクの本質的な難しさ。
