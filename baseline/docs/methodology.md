## 1. タスク定義

### コンペティション概要
- NCAAバスケットボールトーナメント（男子+女子）の勝敗確率を予測する
- 評価指標: **Brier Score**（予測確率と実結果のMSE）
- 提出形式: `2026_{LowerTeamID}_{HigherTeamID}` に対して、LowerTeamIDの勝率を予測
- 全チーム間の仮想対戦（132,133マッチアップ）を予測

### 問題の特性
- 二値分類問題だが、**確率の正確さ**が問われる（勝敗の方向だけでなく確信度も重要）
- 学習データはトーナメント試合のみ（約4,300試合）で比較的小規模
- 男女でデータ構造は同一だが期間・特性が異なる
- 2026年のシード情報は提出時に未確定

## 2. EDA: データの理解

### 2.1 分析の進め方

35個のCSVファイル（合計約721万行）を以下の観点で分析した。

1. **予測に使える情報は何か** — 各ファイルの内容とリレーションを整理
2. **何がトーナメント勝敗を最もよく説明するか** — 統計量との相関分析
3. **男女の違いは何か** — 別々の扱いが必要な箇所の特定
4. **2026年に利用できるデータは何か** — 予測時の制約を把握

### 2.2 シード分析

**結論: シードはトーナメント勝敗の最も強力な予測因子**

シードとは、NCAAトーナメントの組み合わせにおける強さの序列（1位が最強、16位が最弱）を表す。

分析手法: NCAATourneySeeds.csv とトーナメント結果を結合し、シードの組み合わせ別の勝率を集計した。

| マッチアップ | 男子 上位勝率 | 女子 上位勝率 |
|-------------|-------------|-------------|
| 1 vs 16 | 98.8% | ≈100% |
| 2 vs 15 | 93.1% | 100% |
| 5 vs 12 | 64.4% | — |
| 8 vs 9 | 48.1% (ほぼ五分) | — |

**知見**:
- 女子はシードの予測力が男子よりさらに高い（上位勝率76.2% vs 68.9%）
- 8 vs 9は実質コイントスに近い → モデルはこの領域で0.5付近を出すべき
- シードはfeature importanceでも1位を獲得した

**モデルへの反映**: `diff_seed_num`（シード差分）、`seed_product`（積）、`seed_sum`（和）を特徴量として採用。ただし2026年はシード未確定のためNaNになり、LightGBMのNaN処理に委ねる。

### 2.3 Elo分析

**結論: K=30-50, reset_factor=0.75が最適。年度間の安定性が高い。**

分析手法: レギュラーシーズンの試合結果を時系列順に処理し、K値（20, 30, 40, 50）× reset_factor の組み合わせでEloを計算。各設定でトーナメント結果に対するBrier Scoreを算出した。

- K=40-50: Brier=0.192、K=20-30でも大差なし（0.193程度）
- 年度間のElo相関: 平均 **0.861** → チームの強さは年度をまたいで高い一貫性を持つ
- reset_factor=0.75（シーズン開始時に25%を平均に回帰）が効果的

**最終採用**: K=32, margin-of-victory調整あり, reset_factor=0.75

### 2.4 レギュラーシーズン統計の予測力

**結論: 効率性メトリクスと得失点差が最も予測力が高い。3P%やFT%は弱い。**

分析手法: DetailedResultsから各チームのシーズン通算統計を算出し、トーナメント勝利数との相関係数を計算した。

| 統計量 | 相関係数 r | モデルでの扱い |
|--------|-----------|-------------|
| 得失点差 | +0.442 | `diff_margin` として採用 |
| ネット効率 | +0.431 | `diff_net_eff` として採用 |
| 勝率 | +0.343 | `diff_win_pct` として採用 |
| 攻撃効率 | +0.329 | `diff_off_eff` として採用 |
| 3P% | +0.085 | 採用するが重要度は低い |
| FT% | +0.065 | 同上 |

**知見**: 生のスコア（ppg）よりも、ポゼッションあたりの効率指標（off_eff, def_eff）の方が予測力が高い。これはペースの違いを正規化する効果による。

### 2.5 Massey Ordinals分析

**結論: Top 3-10システムのアンサンブルが最適。196全システムを使うと精度低下。**

分析手法: 各ランキングシステム単体でのトーナメント勝敗の的中率を算出し、上位N個の平均ランクでアンサンブルした場合の精度を比較した。

- 上位: POM (71.4%), SAG, MOR, WLK — 数理モデルベースのシステム
- 下位: AP, USA (~61%) — 投票ベースは予測力が低い
- Top 10で71.5%, Top 50で70.7% → **少数精鋭が効果的**

**最終採用**: Top 10システム (POM, SAG, MOR, WLK, DOL, COL, RTH, WOL, CNG, MB) の集約統計 + POM/SAG/MORの個別ランク

### 2.6 ホーム/アウェイ効果

**結論: 女子トーナメントのホーム開催は勝率81.3%で極めて大きな影響。男子トーナメントは全試合ニュートラル。**

| 条件 | 勝率 |
|------|------|
| 男子RS ホーム | 65.8% |
| 男子トーナメント（全てニュートラル） | — |
| 女子トーナメント ホーム | **81.3%** |
| 女子トーナメント ニュートラル | — |

**モデルへの反映**: 現バージョンではトーナメントの開催地情報は直接特徴量に入れていない。Eloの計算時にホームアドバンテージを考慮している（RS: +100, トーナメント: 0）。今後の改善点として女子の開催地特徴量の追加が考えられる。

### 2.7 カンファレンスの強さ

**結論: 間接的に有用。Eloベースのカンファレンス平均強度を特徴量化。**

男子トップ: ACC > Big Ten > Big East > SEC > Big Twelve
女子トップ: SEC > ACC > Big Twelve

**モデルへの反映**: カンファレンス名自体ではなく、所属カンファレンスのElo平均/中央値を数値特徴量として使用。これはカテゴリ変数を使うよりもデータ効率が良く、新しいカンファレンス再編にも頑健。

### 2.8 2026年データの可用性確認

| データ | 2026年で利用可能か | 備考 |
|--------|------------------|------|
| RS結果 (Compact/Detail) | ○ | day 93まで（2月上旬） |
| Massey Ordinals | ○ | 56システム, day 93まで |
| カンファレンス所属 | ○ | |
| シード | **×** | Selection Sunday前のため未確定 |
| トーナメント結果 | **×** | 未開催 |

→ シードに依存しすぎないモデル設計が必要。

## 3. 特徴量エンジニアリング

### 3.1 設計方針

1. **差分ベース**: 全てのチーム特徴量を `feat_A - feat_B` の差分として計算。TeamA = LowerTeamIDとして正規化し、提出形式と一致させる
2. **データリークの防止**: 各シーズンのレギュラーシーズンデータのみから特徴量を構築。トーナメント結果は一切使わない
3. **男女共通パイプライン**: 同じ関数で男女両方の特徴量を構築。男子のみの特徴量（Massey）は自然にNaNになる

### 3.2 特徴量一覧（51個）

#### Elo関連（3個）
| 特徴量 | 算出方法 |
|--------|---------|
| `diff_elo` | TeamAのElo - TeamBのElo |
| `elo_win_prob` | Elo差分からのロジスティック勝率: 1/(1+10^(-diff/400)) |
| (内部) | Eloはトーナメント結果でもシーズン間で更新（キャリーオーバー） |

**Eloの計算詳細**:
```
1. 各シーズン開始時に平均回帰: elo_new = 0.75 * elo_prev + 0.25 * mean_elo
2. 各試合後にEloを更新:
   - 期待勝率: E = 1 / (1 + 10^(-(elo_w + home_adv - elo_l) / 400))
   - MoVマルチプライヤ: mult = ln(margin+1) * 2.2/(|elo_diff|*0.001+2.2)
   - 更新: elo_w += K * mult * (1 - E), elo_l -= K * mult * (1 - E)
3. RS完了後のEloを特徴量として記録
4. トーナメント結果でさらにEloを更新（ただし特徴量には使わず次シーズンに引き継ぐ）
```

Margin-of-Victory (MoV) 調整の狙い: 接戦の勝利と大差の勝利を区別する。大差の場合はElo更新を大きくし、Elo差が大きい場合は更新を抑制する（すでに実力差が反映されているため）。

#### レギュラーシーズン統計（25個の差分）
DetailedResultsがあるシーズン（男子: 2003〜, 女子: 2010〜）ではフル統計を、それ以前はCompactResultsから基本統計のみを算出。

**算出方法**:
1. 各試合を勝者・敗者の両方の視点から2行に展開（例: チームAがチームBに勝利 → チームA: Win=1, Score=75, OppScore=60 / チームB: Win=0, Score=60, OppScore=75）
2. (Season, TeamID) でグループ化し、シーズン通算を集計
3. 効率指標はポゼッション数で正規化: `Poss = FGA - OR + TO + 0.475 * FTA`

| カテゴリ | 特徴量 | 計算式 |
|---------|--------|--------|
| 基本 | win_pct, ppg, opp_ppg, margin | 勝敗・得点の単純平均 |
| 効率 | off_eff, def_eff, net_eff | 100ポゼッションあたりの得点/失点 |
| シューティング | fg_pct, fg3_pct, ft_pct, efg_pct, ts_pct | FGM/FGA等。eFG%は3Pを1.5倍で補正。TS%は全得点機会に対する効率 |
| リバウンド | or_pct, dr_pct | 自チームOR / (自OR + 相手DR) 等 |
| その他 | ast_pg, to_pg, stl_pg, blk_pg, pf_pg, pace | 1試合あたりの各統計 |
| 守備 | opp_fg_pct, opp_fg3_pct | 相手のシューティング効率 |
| 安定性 | score_std, margin_std | 得点・得失点差の標準偏差 |
| 派生 | ast_to_ratio | アシスト/ターンオーバー比 |

#### Massey Ordinals（男子のみ, 10個の差分）
- Top 10システムの集約: mean, median, min, max, std
- 個別システム: POM, SAG, MOR の生ランク
- 全システムの集約: mean, median

各システムについて**シーズン最終日のランキング**を使用。

#### SOS（2個の差分）
- `sos`: 対戦した全相手の平均Elo
- `sos_std`: 同標準偏差（スケジュールの多様性を表す）

#### カンファレンス強度（2個の差分）
- `conf_elo_mean`: 所属カンファレンス全チームの平均Elo
- `conf_elo_median`: 同中央値

#### 直近フォーム（2個の差分）
- `recent_win_pct`: シーズン最後10試合の勝率
- `recent_avg_margin`: 同平均得失点差

#### シード関連（3個）
- `diff_seed_num`: シード番号差分
- `seed_product`: シード番号の積（1×16=16, 8×9=72のように非線形性を捉える）
- `seed_sum`: シード番号の和

#### その他（1個）
- `gender`: 男子=0, 女子=1

### 3.3 欠損値の扱い

LightGBMはNaNをネイティブに処理可能。特に以下のケースで欠損が発生する:
- 2026年のシード情報 → 全チームNaN
- DetailedResultsがないシーズンの統計量（fg_pct等） → NaN
- Massey Ordinals（女子は全てNaN、男子も2003年以前はNaN）
- トーナメント非出場チームのシード → NaN

明示的な補完は行わず、LightGBMの分岐処理に任せた。

## 4. モデル設計

### 4.1 アルゴリズム選択: LightGBM

**選定理由**:
- テーブルデータに対する高い予測性能
- 欠損値のネイティブ処理（シード欠損への対応）
- 学習が高速（CPU環境でも数分以内）
- 特徴量の重要度を自然に出力可能

他の選択肢との比較:
- ロジスティック回帰: 非線形パターン（シードの組み合わせ効果等）を捉えにくい
- ニューラルネット: データ量（4,302試合）に対して過剰。GPU推奨
- XGBoost: LightGBMとほぼ同等だが、欠損値処理でLightGBMがやや有利

### 4.2 ハイパーパラメータ

グリッドサーチ（81パターン: lr × num_leaves × reg_lambda × min_child_samples）で最適化した。

```python
{
    'learning_rate': 0.02,     # 低い学習率で安定した学習
    'num_leaves': 8,           # 浅い木で過学習を抑制
    'max_depth': 5,
    'min_child_samples': 30,   # 各リーフの最小サンプル数
    'subsample': 0.7,          # 行のサブサンプリング
    'colsample_bytree': 0.7,   # 列のサブサンプリング
    'reg_alpha': 1.0,          # L1正則化
    'reg_lambda': 10.0,        # L2正則化（強め）
}
```

**設計意図**: 4,302試合というデータ量に対して51特徴量は比較的多い。過学習リスクが高いため、浅い木 (num_leaves=8)、強い正則化 (reg_lambda=10.0)、サブサンプリング (0.7) で積極的に抑制した。学習率は低め (0.02) にして、early stoppingで最適なラウンド数 (375) を決定した。

### 4.3 アンサンブル

5つのモデルを以下の多様性を持たせて学習:
- 異なるランダムシード
- `colsample_bytree`: 0.6, 0.7, 0.8 のローテーション
- `subsample`: 0.65, 0.70, 0.75 のローテーション

予測値の単純平均を最終予測とする。アンサンブルの効果は小さいが（Brier 0.1705→0.1707）、予測の分散を低減する効果がある。

### 4.4 Eloブレンド

```
final_pred = 0.85 * lgbm_ensemble + 0.15 * elo_win_prob
```

**狙い**: LightGBMの予測にEloの安定した勝率推定を混ぜることで、特にシード情報がないマッチアップ（2026年の大部分）でのロバスト性を向上させる。Eloは単体でBrier≈0.192の性能を持つベースライン予測器である。

## 5. 学習パイプライン

### 5.1 全体フロー

```
[データ読み込み]
      ↓
[チーム特徴量構築]           ← Elo, RS統計, Massey, Seeds, SOS, Conference, Form
  男子: 13,753行 × 45列
  女子:  9,851行 × 35列
      ↓
[対戦ペア構築]               ← トーナメント結果 → (Season, TeamA, TeamB, label)
  4,302試合
      ↓
[差分特徴量計算]             ← feat_A - feat_B + 追加特徴量
  51特徴量
      ↓
[学習・バリデーション]       ← Train: ~3,637 / Val: 665
      ↓
[全データで再学習 × 5]      ← アンサンブル
      ↓
[2026予測・提出ファイル生成]  ← 132,133マッチアップ
```

### 5.2 学習データの構築

トーナメント結果から学習データを構築する手順:
1. `NCAATourneyCompactResults.csv` から (WTeamID, LTeamID) を取得
2. `TeamA = min(WTeamID, LTeamID)`, `TeamB = max(WTeamID, LTeamID)` として正規化
3. `label = 1 if WTeamID == TeamA else 0` — TeamAが勝ったかどうか
4. チーム特徴量テーブルと (Season, TeamA/B) でmergeし、差分を計算

**label分布**: 約50:50（TeamIDが小さいチームが勝つ確率はほぼ半々。これは提出形式の正規化による自然な結果）

### 5.3 データリーク防止

以下の原則を厳守:
- **Eloの特徴量はRS終了時点のもの**: トーナメント結果はEloの更新には使うが、同一シーズンの特徴量には反映しない
- **シード情報は同一シーズンから取得**: 未来のシーズンの情報は使わない
- **バリデーションは時系列分割**: 2021-2025をテスト、それ以前を学習。ランダム分割による時間方向のリークを防止

## 6. バリデーションと結果

### 6.1 バリデーション戦略

- **時系列分割**: 2021-2025年の5シーズンをテスト、それ以前の全シーズンを学習
- 2020年はCOVIDでトーナメント中止のため、データに含まれない
- シーズン単位の分割は、実際のタスク（過去のデータで未来のトーナメントを予測）を忠実に模倣

### 6.2 最終結果

| 指標 | Single Model | Ensemble (5 models) |
|------|-------------|-------------------|
| **Brier Score** | **0.1705** | 0.1707 |
| Log Loss | 0.506 | 0.507 |
| Accuracy | 74.4% | **74.9%** |

### 6.3 シーズン別結果

| Season | Brier | Accuracy | N | 備考 |
|--------|-------|----------|---|------|
| 2021 | 0.1890 | 70.5% | 129 | COVID明け、変則シーズン |
| 2022 | 0.1852 | 71.6% | 134 | |
| 2023 | 0.1897 | 74.6% | 134 | |
| 2024 | 0.1600 | 73.9% | 134 | アップセット少なめ |
| 2025 | 0.1302 | 83.6% | 134 | 予測しやすい年 |

シーズン間のBrierの幅 (0.13-0.19) はトーナメントの固有のランダム性（アップセット頻度の年次変動）に起因する。

### 6.4 Feature Importance

| 順位 | 特徴量 | Gain | 解釈 |
|-----|--------|------|------|
| 1 | diff_seed_num | 16,289 | シード差 — 圧倒的に最重要 |
| 2 | diff_elo | 8,558 | Eloレーティング差 |
| 3 | elo_win_prob | 7,101 | Eloからのロジスティック勝率 |
| 4 | diff_sos | 3,047 | 対戦相手の平均Elo差 |
| 5 | diff_margin | 2,574 | 平均得失点差の差 |
| 6 | diff_conf_elo_mean | 647 | カンファレンス強度差 |
| 7 | diff_conf_elo_median | 540 | 同上（中央値） |
| 8 | diff_sos_std | 382 | スケジュール多様性差 |
| 9 | diff_recent_avg_margin | 340 | 直近10試合の得失点差 |
| 10 | diff_to_pg | 333 | ターンオーバー差 |

**考察**: 上位5つだけでimportanceの大半を占める。シード > Elo > SOS > margin の順に予測力が強い。統計量の中ではmargin（得失点差）が最も有用で、個別のbox score統計（FG%, 3P%等）は相対的に重要度が低い。これはEDAの相関分析と一致する。

## 7. 予測と提出ファイル生成

### 7.1 手順

1. `SampleSubmissionStage2.csv` (132,133行) をパース: `{Season}_{TeamA}_{TeamB}` 形式
2. TeamIDで男女を分割: TeamA < 3000 なら男子、3000以上なら女子
3. 各マッチアップに対して、学習時と同じ特徴量構築パイプラインを適用
4. 5つのモデルの予測平均 + Eloブレンド
5. [0.02, 0.98] にクリッピング

### 7.2 2026年の特殊事情

- **シード**: 全チームNaN（Selection Sunday前のため）→ diff_seed_num, seed_product, seed_sum が全てNaN
- **Massey Ordinals**: day 93時点のデータが利用可能（56システム）
- **RS統計**: day 93までの試合データが利用可能
- **トーナメント非出場チームも予測対象**: 全365チーム（男子）× 全チーム のペアを予測

### 7.3 予測値の分布


## 8. 開発の反復過程

### v1 → v2 → v3 の改善履歴

| バージョン | Brier | 主な変更 |
|-----------|-------|---------|
| v1 | 0.1762 | 基本的なElo + RS統計 + Massey + Seed。行単位のループ処理 |
| v2 | 0.1715 | ベクトル化による高速化。SOS, 直近フォーム追加。正則化強化 |
| v3 | 0.1705 | Eloのトーナメントキャリーオーバー。カンファレンス強度。アンサンブル。ハイパーパラメータ最適化 |

### 各バージョンで何が効いたか

**v1 → v2 (Brier: 0.1762 → 0.1715, -0.0047)**
- SOS（対戦相手強度）の追加が大きく寄与
- 正則化強化（reg_lambda: 1.0→5.0, num_leaves: 31→16）で過学習を軽減
- Log Loss: 0.523 → 0.508 と大幅改善

**v2 → v3 (Brier: 0.1715 → 0.1705, -0.0010)**
- Eloのトーナメントキャリーオーバー: 前年トーナメントでの結果が次シーズンのEloに反映
- カンファレンス強度: importance #6-7に入る有用な特徴量
- elo_win_prob: Eloの差分を直接勝率に変換した特徴量が#3の重要度
- ハイパーパラメータ最適化: num_leaves: 16→8, reg_lambda: 5.0→10.0
- アンサンブル: 分散低減（Accuracyで+0.5%改善）

### 試みたが効果が限定的だったもの

- **Eloブレンド** (15%): Brier改善は0.0001程度。モデルが既にEloを特徴量として使っているため追加効果は小さい
- **3P%, FT%の単体特徴量**: EDAで相関が弱いことが示されており、feature importanceでも下位
- **アンサンブルモデル数を増やす**: 5モデルで十分。それ以上は計算コストに見合わない
